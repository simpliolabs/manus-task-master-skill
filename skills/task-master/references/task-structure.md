## Task JSON Schema

Tasks are stored in `.taskmaster/tasks/tasks.json`. Example:

```json
{
  "tasks": [
    {
      "id": 1,
      "title": "Set up project infrastructure",
      "description": "Initialize the project with required tooling and configuration",
      "status": "done",
      "priority": "high",
      "dependencies": [],
      "details": "Use Vite for bundling, ESLint for linting, Prettier for formatting...",
      "testStrategy": "Verify build succeeds, linting passes, and dev server starts",
      "subtasks": [
        {
          "id": 1,
          "title": "Configure Vite build system",
          "description": "Set up Vite with React and TypeScript plugins",
          "status": "done",
          "dependencies": [],
          "details": "Install vite, @vitejs/plugin-react, configure vite.config.ts..."
        }
      ]
    }
  ]
}
```

**Never edit this file manually.** Always use CLI commands or MCP tools.

## Task ID Format

| Format | Example | Description |
|--------|---------|-------------|
| Integer | `1`, `15` | Top-level task |
| Dot notation | `1.2`, `15.3` | Subtask (parent.child) |
| Sub-subtask | `1.1.2` | Nested subtask |
| Comma-separated | `1,3,5` | Multiple tasks (for show/status) |
| Mixed | `44,44.1,55.2` | Mix of tasks and subtasks |

## Task Fields

| Field | Type | Description |
|-------|------|-------------|
| `id` | number | Unique task identifier |
| `title` | string | Brief descriptive title |
| `description` | string | Concise summary of what the task involves |
| `status` | string | Current state (see Status Values) |
| `priority` | string | `high`, `medium`, or `low` |
| `dependencies` | array | IDs of prerequisite tasks (e.g., `[1, 2.1]`) |
| `details` | string | In-depth implementation instructions |
| `testStrategy` | string | How to verify the task is complete |
| `subtasks` | array | Child tasks with same structure (minus subtasks) |

Dependencies display with status indicators: `✅` for completed, `⏱️` for pending.

## Status Values

| Status | Meaning |
|--------|---------|
| `pending` | Ready to work on |
| `in-progress` | Currently being worked on |
| `done` | Completed and verified |
| `deferred` | Postponed for later |
| `cancelled` | No longer needed |
| `blocked` | Waiting on external factors |

Setting a parent to `done` auto-marks all subtasks as `done`. Prefer `deferred`/`cancelled`/`blocked` over `remove-task` to preserve history.

## Dependencies

Dependencies enforce execution order. A task cannot start until all its dependencies are `done`.

```bash
task-master add-dependency --id=5 --depends-on=3      # Task 5 depends on task 3
task-master remove-dependency --id=5 --depends-on=3   # Remove dependency
task-master validate-dependencies                      # Check for issues
task-master fix-dependencies                           # Auto-fix problems
```

The system prevents circular dependencies and duplicate entries. Dependencies are auto-cleaned when tasks are removed or moved.

## Configuration Files

### `.taskmaster/config.json`

Managed via `task-master models` commands. Structure:

```json
{
  "models": {
    "main": {
      "provider": "anthropic",
      "modelId": "claude-3-7-sonnet-20250219",
      "maxTokens": 64000,
      "temperature": 0.2
    },
    "research": {
      "provider": "perplexity",
      "modelId": "sonar-pro",
      "maxTokens": 8700,
      "temperature": 0.1
    },
    "fallback": {
      "provider": "anthropic",
      "modelId": "claude-3-5-sonnet",
      "maxTokens": 64000,
      "temperature": 0.2
    }
  },
  "global": {
    "logLevel": "info",
    "defaultNumTasks": 10,
    "defaultSubtasks": 5,
    "defaultPriority": "medium",
    "defaultTag": "master",
    "projectName": "My Project"
  }
}
```

### `.taskmaster/state.json`

Tracks tag state. Auto-managed, do not edit:

```json
{
  "currentTag": "master",
  "lastSwitched": "2025-06-11T20:26:12.598Z",
  "migrationNoticeShown": true
}
```

### `.env` (API Keys)

```
ANTHROPIC_API_KEY=sk-ant-...
PERPLEXITY_API_KEY=pplx-...
OPENAI_API_KEY=sk-...
GOOGLE_API_KEY=AIzaSy...
```

Only API keys belong in `.env`. All other settings go in `config.json`.

## Complexity Reports

Generated by `task-master analyze-complexity --research`. Stored at `.taskmaster/reports/task-complexity-report.json`.

Reports include per-task complexity scores (1-10) and recommended subtask counts. The `expand` command automatically uses these recommendations when available.

Focus on tasks scoring 8-10 for detailed breakdown. Use `task-master complexity-report` for a formatted view.
